# Traffic CCTV Database Models

This repository defines a relational schema and SQLAlchemy ORM models for managing:

- Intersections
- Streets
- CCTVs
- Object detections
- Polygonal regions and their points
- Mappings of detections to regions

It is intended to be used as a backend data layer (e.g., for a FastAPI or Flask service)

---

## Data Model Overview

### 1. Intersection

**Table**: `intersections`

Represents a traffic intersection.

Columns:

- `id` (PK, int, autoincrement)
- `name` (str, required)
- `time` (datetime, server default `NOW()`)

Relationships:

- `streets`: One-to-many → `Street`
- `cctvs`: One-to-many → `CCTV`

Cascades: Deleting an `Intersection` deletes associated `streets` and `cctvs`.

---

### 2. Street

**Table**: `streets`

Represents a street that belongs to an intersection.

Columns:

- `id` (PK, int, autoincrement)
- `intersection_id` (FK → `intersections.id`, required, `ON DELETE CASCADE`)
- `name` (str, required)
- `time` (datetime, server default `NOW()`)

Relationships:

- `intersection`: Many-to-one → `Intersection`
- `regions`: One-to-many → `Region`

Cascades: Deleting a `Street` deletes associated `regions`.

---

### 3. CCTV

**Table**: `cctvs`

Represents a CCTV camera located at an intersection.

Columns:

- `id` (PK, int, autoincrement)
- `intersection_id` (FK → `intersections.id`, required, `ON DELETE CASCADE`)
- `name` (str, required)
- `time` (datetime, server default `NOW()`)

Relationships:

- `intersection`: Many-to-one → `Intersection`
- `detections`: One-to-many → `Detection`
- `regions`: One-to-many → `Region`

Cascades: Deleting a `CCTV` deletes its `detections` and `regions`.

---

### 4. Detection

**Table**: `detections`

Represents an object detection from a CCTV frame.

Columns:

- `id` (PK, int, autoincrement)
- `cctv_id` (FK → `cctvs.id`, required, `ON DELETE CASCADE`)
- `x` (int, required) – X coordinate (e.g., pixel)
- `y` (int, required) – Y coordinate (e.g., pixel)
- `type` (str, required) – e.g., `car`, `person`, `bike`
- `time` (datetime, server default `NOW()`)

Relationships:

- `cctv`: Many-to-one → `CCTV`
- `detections_in_regions`: One-to-many → `DetectionInRegion`

Cascades: Deleting a `Detection` deletes associated `detections_in_regions`.

---

### 5. Region

**Table**: `regions`

Represents a polygonal region on a camera view, associated with both a CCTV and a street (e.g., a lane or area of interest).

Columns:

- `id` (PK, int, autoincrement)
- `cctv_id` (FK → `cctvs.id`, required, `ON DELETE CASCADE`)
- `street_id` (FK → `streets.id`, required, `ON DELETE CASCADE`)
- `time` (datetime, server default `NOW()`)

Relationships:

- `cctv`: Many-to-one → `CCTV`
- `street`: Many-to-one → `Street`
- `region_points`: One-to-many → `RegionPoint` (polygon vertices)
- `detections_in_regions`: One-to-many → `DetectionInRegion`

Cascades: Deleting a `Region` deletes its `region_points` and `detections_in_regions`.

---

### 6. RegionPoint

**Table**: `region_points`

Represents a single vertex of a region polygon.

Columns:

- `id` (PK, int, autoincrement)
- `region_id` (FK → `regions.id`, required, `ON DELETE CASCADE`)
- `x` (int, required)
- `y` (int, required)
- `time` (datetime, server default `NOW()`)

Relationships:

- `region`: Many-to-one → `Region`

---

### 7. DetectionInRegion

**Table**: `detections_in_regions`

Associative table linking a detection to a specific region (i.e., which detection occurred in which region and when).

Columns:

- `id` (PK, int, autoincrement)
- `region_id` (FK → `regions.id`, required, `ON DELETE CASCADE`)
- `detection_id` (FK → `detections.id`, required, `ON DELETE CASCADE`)
- `time` (datetime, server default `NOW()`)

Relationships:

- `region`: Many-to-one → `Region`
- `detection`: Many-to-one → `Detection`

---

## Notes

- All `time` columns use `server_default=func.now()`; the timestamps are generated by the database server.
- Most foreign keys use `ON DELETE CASCADE`, and relationships use `cascade="all, delete"`, ensuring that related child rows are automatically removed when their parent is deleted (both at DB level and ORM level).
